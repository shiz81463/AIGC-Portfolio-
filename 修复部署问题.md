# 修复 Vercel 部署错误

## 错误信息
```
Failed to resolve /src/main.jsx from /vercel/path0/index.html
```

## 解决方案

### 步骤 1：确保所有文件都已提交到 GitHub

在终端运行以下命令，确保所有文件都已推送到 GitHub：

```bash
cd /Users/ziconghuang/portfolio-showcase

# 检查哪些文件还没有提交
git status

# 添加所有文件
git add .

# 提交更改
git commit -m "修复部署配置"

# 推送到 GitHub
git push
```

### 步骤 2：验证文件结构

确保以下文件都在 GitHub 仓库中：
- ✅ `index.html` (在根目录)
- ✅ `src/main.jsx` (在 src 文件夹中)
- ✅ `src/App.jsx` (在 src 文件夹中)
- ✅ `package.json` (在根目录)
- ✅ `vite.config.js` (在根目录)
- ✅ `public/` 文件夹及其所有文件

### 步骤 3：在 Vercel 中重新部署

1. 登录 Vercel
2. 进入您的项目
3. 点击 "Redeploy"（重新部署）按钮
4. 选择最新的 commit
5. 点击 "Redeploy"

### 步骤 4：如果还是失败，检查 Vercel 项目设置

在 Vercel 项目设置中，确保：

1. **Framework Preset**: Vite
2. **Build Command**: `npm run build`
3. **Output Directory**: `dist`
4. **Install Command**: `npm install`
5. **Root Directory**: `./` (留空或使用 `./`)

### 步骤 5：查看完整错误日志

在 Vercel 部署页面：
1. 点击 "Build Logs"（构建日志）
2. 展开查看完整错误信息
3. 检查是否有其他错误提示

## 常见问题

### 问题：文件没有推送到 GitHub
**解决**：运行 `git add .` 和 `git push` 确保所有文件都在 GitHub 上

### 问题：Node 版本不匹配
**解决**：我已经在 `package.json` 中添加了 `"engines": { "node": ">=18.0.0" }`

### 问题：路径解析错误
**解决**：我已经更新了 `vite.config.js` 配置文件
