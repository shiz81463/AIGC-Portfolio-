# 作品集网站部署详细步骤

按照以下步骤，将您的作品集网站发布到互联网上。

---

## 第一步：在项目中初始化 Git

打开终端（Terminal），运行以下命令：

```bash
cd /Users/ziconghuang/portfolio-showcase
git init
git add .
git commit -m "初始提交 - 作品集网站"
```

**作用说明：** 在项目文件夹中设置 Git 版本控制。

---

## 第二步：在 GitHub 上创建仓库

1. **访问 GitHub.com** 并登录您的账户
2. **点击右上角的 "+" 图标**
3. **点击 "New repository"（新建仓库）**
4. **填写仓库信息：**
   - Repository name（仓库名称）：`portfolio-showcase`（或您喜欢的任何名称）
   - Description（描述）："我的 AIGC 设计师作品集"
   - 选择：**Public（公开）**（这样 Vercel 才能访问）
   - **不要勾选** "Initialize with README"（我们已经有了文件）
5. **点击 "Create repository"（创建仓库）**

**您会看到一个设置说明页面 - 先不要按照那个操作！**

---

## 第三步：将本地项目连接到 GitHub

回到终端，运行以下命令（将 `YOUR_USERNAME` 替换为您的实际 GitHub 用户名）：

```bash
cd /Users/ziconghuang/portfolio-showcase
git remote add origin https://github.com/YOUR_USERNAME/portfolio-showcase.git
git branch -M main
git push -u origin main
```

**示例：** 如果您的 GitHub 用户名是 `alexhuang`，命令应该是：
```bash
git remote add origin https://github.com/alexhuang/portfolio-showcase.git
```

**执行过程：**
- 第一条命令：将本地文件夹连接到 GitHub
- 第二条命令：将分支重命名为 "main"
- 第三条命令：将所有文件上传到 GitHub

**注意：** 系统会要求您输入 GitHub 用户名和密码（如果启用了双重验证，需要使用个人访问令牌）

---

## 第四步：验证文件已上传到 GitHub

1. **返回您的 GitHub 仓库页面**（如果需要，请刷新）
2. **您应该能看到所有文件：**
   - `src/` 文件夹
   - `public/` 文件夹
   - `package.json`
   - `index.html`
   - 等等

**如果您看到所有文件，就可以进行下一步了！**

---

## 第五步：部署到 Vercel

1. **访问 https://vercel.com** 并登录
2. **点击 "Add New..." 按钮**（或 "New Project"）
3. **点击 "Import Git Repository"（导入 Git 仓库）**
4. **您会看到 GitHub 仓库列表**
5. **找到并点击 `portfolio-showcase`**（或您命名的名称）
6. **点击 "Import"（导入）**

---

## 第六步：配置 Vercel 项目设置

在 "Configure Project"（配置项目）页面：

1. **Project Name（项目名称）：** 保持为 `portfolio-showcase`（或根据需要更改）
2. **Framework Preset（框架预设）：** 应该自动检测为 "Vite" - **保持默认**
3. **Root Directory（根目录）：** 保持为 `./`（默认）
4. **Build Command（构建命令）：** 应该是 `npm run build` - **保持默认**
5. **Output Directory（输出目录）：** 应该是 `dist` - **保持默认**
6. **Install Command（安装命令）：** 应该是 `npm install` - **保持默认**

**点击底部的 "Deploy"（部署）按钮**

---

## 第七步：等待部署完成

您会看到一个进度屏幕，显示：
- "Installing dependencies..."（正在安装依赖...）
- "Building..."（正在构建...）
- "Deploying..."（正在部署...）

**这个过程大约需要 1-2 分钟。** 请耐心等待！

---

## 第八步：您的网站已上线！🎉

部署完成后，您会看到：

1. **"Congratulations!"（恭喜！）消息**
2. **一个类似这样的 URL：** `https://portfolio-showcase-xxxxx.vercel.app`
3. **点击 "Visit"（访问）** 查看您的在线作品集！

**您的作品集现在所有人都可以访问了！**

---

## 第九步：分享您的作品集

复制您的 Vercel URL 并分享：
- 放在您的电子邮件签名中
- 添加到您的 LinkedIn 个人资料
- 与潜在客户/雇主分享

**URL 格式示例：**
```

```

---

## 未来更新（如何更新您的网站）

每当您对作品集进行更改时：

1. **在代码中进行更改**
2. **提交并推送到 GitHub：**
   ```bash
   cd /Users/ziconghuang/portfolio-showcase
   git add .
   git commit -m "更新作品集"
   git push
   ```
3. **Vercel 会自动重新部署** - 您的网站在 1-2 分钟内更新！

**无需再次手动部署 - 这是自动的！**

---

## 故障排除

### 如果 Git 要求输入凭据：
- 使用您的 GitHub 用户名
- 对于密码，使用**个人访问令牌**（不是您的密码）
- 创建令牌：GitHub → Settings（设置） → Developer settings（开发者设置） → Personal access tokens（个人访问令牌） → Generate new token（生成新令牌）

### 如果 Vercel 找不到您的仓库：
- 确保仓库是**公开的**（不是私有的）
- 尝试刷新 Vercel 页面

### 如果构建失败：
- 检查所有文件是否已推送到 GitHub
- 确保 `package.json` 在根文件夹中
